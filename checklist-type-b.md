# Checklist Type B

---

### Пункт 1: В коде нет никаких комментариев на русском языке

**Тип:** B (Простая проверка через grep)  
**Описание:** В коде нет никаких комментариев на русском языке

**Особые детали:**

- Проверить комментарии `//`, `/*`, `/**` (JSDoc)

**Алгоритм проверки:**

1. Использовать grep для поиска кириллицы в изменённых файлах:

```bash
"grep -r "[А-Яа-яЁё]" src/ --include="*.ts""
```

2. Искать только в изменённых файлах из коммита.
3. Проверить найденные строки на наличие в комментариях.

---

### Пункт 2: Не должно быть any типов

**Тип:** B (Простая проверка через grep)  
**Описание:** Не должно быть `any` типов

**Особые детали:**

- Исключение: `@Payload() payload: Record<string, any>`

**Алгоритм проверки:**

1. Использовать grep для поиска `\bany\b` в изменённых файлах.
2. Исключить строки с указанным исключением.
3. Проверить только изменения из коммита.

---

### Пункт 3: Не должно быть console.log в продакшене

**Тип:** B (Простая проверка через grep)  
**Описание:** Не должно быть `console.log` в продакшене

**Особые детали:**

- Исключение: `console.error('error.decoding.cursor.failed:', error);`

**Алгоритм проверки:**

1. Использовать grep для поиска `console\.log` в изменённых файлах.
2. Проверить найденные вхождения.
3. Исключить допустимый `console.error`.

---

### Пункт 4: uuid импорты для генерации должны проходить так: `import { v7 as uuidv7 } from 'uuid'`

**Тип:** B (Простая проверка через grep)  
**Описание:** uuid нужно импортировать только из `uuid` с алиасом `v7 as uuidv7`

**Алгоритм проверки:**

1. Использовать grep для поиска импортов uuid:

```bash
"grep -r "from ['\"]uuid" src/ --include="*.ts""
```

2. Проверить формат импорта: `from 'uuid'` и `v7 as uuidv7`.
3. Убедиться, что нет `from 'uuidv7'` и других вариантов.

---

### Пункт 5: Все логи должны быть в формате lowercase с точками вместо пробелов

**Тип:** B (Простая проверка через grep)  
**Описание:** Логи должны выглядеть как `user.created.successfully: ${userId}`

**Особые детали:**

- только строчные буквы;
- слова разделены точками;
- переменные в конце отделены двоеточием;
- разрешается искать по части строки.

**Алгоритм проверки:**

1. Использовать grep для поиска строк логов в изменённых файлах.
2. Проверить формат (lowercase, точки, двоеточие перед переменными).

---

### Пункт 6: В классах с мутациями слово `Resolver` допускается ТОЛЬКО в `@Resolver` и его импорте

**Тип:** B (Простая проверка через grep, только для GraphQL)  
**Описание:** В файлах с `@Mutation` слово `Resolver` не должно встречаться вне импорта и декоратора

**Алгоритм проверки:**

1. Найти все файлы `*/mutations/*.ts`, содержащие `@Mutation(`.
2. В каждом файле выполнить поиск `\bresolver\b` с флагом `-i`.
3. Оставить только строки:
   - `import { ..., Resolver, ... } from '@nestjs/graphql'`
   - `@Resolver(...)`
4. Любые другие вхождения считать нарушением.
