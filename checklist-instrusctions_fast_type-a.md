# Инструкции fast — Тип A (Глобальные проверки)

1. **Подтверждение инструкций.** Перед началом работы выведи блок  
   `Подтверждение инструкций — [./.cursor/checklist-instrusctions_fast_type-a.md#L1](./.cursor/checklist-instrusctions_fast_type-a.md#L1)`  
   и явно укажи, что этот файл прочитан полностью.
2. **Язык.** Все ответы пользователю только на русском.
3. **Подготовка:**
   - определи параметры команды `check` (commits/files/type/instructions/detailedReport/strict/autofix);
   - получи список изменённых файлов командой `git log -n <COMMITS_COUNT> --name-only --pretty=format:` без пустых строк, сохрани порядок;
   - определи интерфейс сервиса (REST/GraphQL) сразу, чтобы помечать неприменимые пункты;
   - загрузить чеклист `./.cursor/checklist-type-a.md` только после выполнения двух шагов выше.
4. **Работай последовательно.** Выполняй все пункты типа A до конца, прежде чем переходить к типу B или C. Не открывай их инструкции и чеклисты заранее.
5. **Формат отчёта для каждого пункта:**
   ```
   Проверяю пункт N (тип A) — [./.cursor/checklist-type-a.md#L<номер>](./.cursor/checklist-type-a.md#L<номер>)
   Запускаю команду `<command>`
   Результат: …
   Статус: ✅/❌/⏭️ (причина)
   ```
   Если нужно несколько команд — добавляй по отдельной строке «Запускаю команду…» и фиксируй результат после каждой.
6. **Команды:** перед любым `run_terminal_cmd` / `read_lints` / `read_file` выводи строку `Запускаю команду \`...\``. После выполнения кратко опиши результат (даже если вывода нет).
7. **Финальный итог по типу A:** перечисли статусы всех пунктов «✅/❌/⏭️» и опиши найденные проблемы либо подтвердите отсутствие нарушений.

> После завершения всех пунктов типа A можно переходить к инструкциям для следующего типа, указанного в параметре `type`.
