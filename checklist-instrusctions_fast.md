ОСНОВНЫЕ ПРАВИЛА

1. Все ответы пользователю — только на русском языке.
2. Перед любыми действиями выведи блок «Подтверждение инструкций», указав, что этот файл прочитан полностью.
3. Не делай предположений. Если данных нет — уточни или пометь пункт как неприменимый.

ОБЩИЙ ПОРЯДОК

1. Определи параметры запуска команды `check` (включая новый параметр `type`, по умолчанию `A`). Параметр может содержать одно значение (`A`) или несколько через запятую (`A,B,C`). Последовательность строгая: сначала A, затем B, затем C. Нельзя переходить к следующему типу, пока текущий не завершён.
2. Получи список изменённых файлов:
   - `git log -n <COMMITS_COUNT> --name-only --pretty=format:` без пустых строк;
   - сохрани порядок — он понадобится типу C.
3. Определи интерфейс сервиса (REST/GraphQL), чтобы заранее знать, какие пункты неприменимы.
4. Для каждого типа из параметра `type`:
   - прочитай соответствующий инструкционный файл полностью (и зафиксируй подтверждение);
   - используй привязанный чеклист;
   - выполни все пункты данного типа;
   - только после этого переходи к следующему типу.

СООТВЕТСТВИЕ ТИПОВ ФАЙЛАМ

| Тип | Инструкции                                         | Чеклист                         |
| --- | -------------------------------------------------- | ------------------------------- |
| A   | `./.cursor/checklist-instrusctions_fast_type-a.md` | `./.cursor/checklist-type-a.md` |
| B   | `./.cursor/checklist-instrusctions_fast_type-b.md` | `./.cursor/checklist-type-b.md` |
| C   | `./.cursor/checklist-instrusctions_fast_type-c.md` | `./.cursor/checklist-type-c.md` |

ОТЧЁТНОСТЬ

- Для detailedReport=1 необходимо использовать детальные блоки, описанные в инструкциях соответствующего типа.
- Для detailedReport=0 допускается краткий отчёт по типам A/B, но для типа C всё равно требуются подробные блоки по каждому пункту.
- Всегда указывай фактические команды (включая пути файлов) и результат их выполнения.

ОПТИМИЗАЦИЯ

1. Фиксируй общий план (подготовка → тип A → тип B → тип C).
2. Не открывай инструкции или чеклист следующего типа до завершения текущего.
3. Если файл очевидно не подходит под конкретный пункт, сразу ставь статус ⏭️ с объяснением.
4. При обнаружении проблемы исправляй её сразу (если `autofix=1`) и повторно выполняй только затронутый пункт.
